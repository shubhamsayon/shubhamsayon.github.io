DOCKERFILE

Dockerfile is a simple text file composed of various arguements and commands listed successively to automatically perform actions on a base image in order to create a new one. Docker can build images automatically by reading the instructions from a Docekrfile.

Points to remember about Dockerfile

  • sequential set of instructions for the Docker engine
  • Order of instructions in dockerfile is very important
  • Each instruction creates a layer and these layers can be reused later i.e. if two docker files need the same layer docker daemon can reuse the precreated layer.
  • Dckerfile has no extension

The Intructions in a Dockerfile can be categorized as:

FUNDAMENTAL INSTRUCTIONS FROM, ARG
CONFIGURATION INTSRUCTIONS RUN,ADD,COPY,ENV
EXECUTION INSTRUCTIONS CMD,EXPOSE
